# 审计汇报规范

## 适用范围

当 AI 完成以下任何操作后，必须附带审计报告：

- 执行 `tasks.md` 中的单条任务（execute-task 第四步）
- 批量实现多条任务
- 完成 `create-proposal` 后置检查
- 任何产出代码文件变更的操作

## 核心约束（NON-NEGOTIABLE）

1. **禁止虚假声明**：禁止在未使用 Read 工具实际读取某文件时声称「已加载」「已读取」该文件。只有工具调用记录中存在对应 Read 操作的，才可标记为「已 Read」。
2. **禁止笼统表述**：禁止用「遵循了团队规范」「按规范执行」等笼统措辞替代具体规范条目引用。必须指明规范文件名 + 具体章节/表格/条目。
3. **明确区分读取状态**：「已 Read」与「未 Read，行为上对齐」是两种不同状态，必须如实标注，不可混淆。
4. **偏差必须披露**：凡与技能定义的步骤不一致之处（如跳过确认、跳过 TDD、批量执行等），必须在报告中明确写出，不可隐瞒。

## 审计报告模板

每次审计报告包含以下五个区块，按顺序输出：

### 区块 1：读取记录

列出本次操作中涉及的所有规范文件和技能文件，标注实际读取状态。

| 类型 | 文件路径 | 状态 |
|------|---------|------|
| 技能 | `.agents/skills/xxx/SKILL.md` | 已 Read / 未 Read |
| 规则 | `.agents/rules/xxx.md` | 已 Read / 未 Read，依据 xxx 推断 |
| 设计 | `openspec/changes/xxx/design.md` | 已 Read / 未 Read |
| 任务 | `openspec/changes/xxx/tasks.md` | 已 Read |

状态取值说明：

- **已 Read**：本次对话中使用 Read 工具读取过该文件
- **未 Read，依据 X 推断**：未直接读取，但依据其它已读文件（如 tasks.md、design.md）中的描述推断行为
- **未 Read，凭既有知识**：未读取，依据训练知识或上下文惯例行事

### 区块 2：操作记录

按时间顺序列出所有工具调用，包括文件操作和终端命令。

格式：`序号. 工具名: 文件路径或命令（操作说明）`

示例：

```
1. Read: package.json
2. Write: src/services/auth.ts（新建认证服务模块）
3. StrReplace: src/router/index.ts（添加 /auth 路由入口）
4. Shell: npm install <dependency>
```

### 区块 3：规范对齐

列出代码中的关键决策点，以及其对应的具体规范条目。

| 操作 | 对应规范 | 具体条目 |
|------|---------|---------|
| 模块目录命名 kebab-case | `02-编码规范.md` | 「命名规范」目录/模块一栏 |
| 接口封装统一通过服务层 | 各 profile 的 API/接口规范 | 「接口请求规范」节 |
| 新增功能附带测试用例 | 各 profile 的测试规范 | 「质量门禁」节 |

若无明确规范对应（如纯业务逻辑），可写「无对应规范，按业务需求实现」。

### 区块 4：技能执行状态

列出本次操作涉及或应涉及的技能，标注执行状态。

| 技能 | 状态 | 备注 |
|------|------|------|
| execute-task | 已 Read，按四步循环执行 | — |
| create-component | 未 Read，行为上对齐 | tasks.md 已写明组件结构 |
| theme-variables | 未涉及 | 本次任务不涉及样式 |

### 区块 5：与规范/技能不一致处

列出所有偏离规范或技能要求的地方，必须说明原因。

示例：

- execute-task 要求逐步确认，本回合用户指示批量执行，未逐步询问
- TDD RED 阶段：项目未配置测试框架，改用类型断言兜底
- 未执行双重审查中的「架构对齐」，因本次仅修改配置文件

若完全一致，写「无偏差」。

## 简化规则

为避免审计报告过于冗长影响效率，以下场景可使用精简版：

| 场景 | 精简方式 |
|------|---------|
| 单条简单任务（如修改一个文件） | 区块 2 和区块 3 可合并为一行 |
| 连续执行同类任务（如连续创建多个 API） | 多条任务共用一份报告，按任务编号分列 |
| 非行为性变更（拼写、注释、格式） | 可省略本规范（同 12-Superpowers执行规范 的跳过条件） |

- `.agents/skills/execute-task/SKILL.md` — Superpowers 四步循环（第四步引用本规范）
