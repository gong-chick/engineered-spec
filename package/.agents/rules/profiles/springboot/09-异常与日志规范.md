# 异常与日志规范（Spring Boot）

## 异常体系

- 定义统一的业务异常基类（如 `BusinessException`）
- 不同业务模块可扩展自己的子类（如 `OrderNotFoundException`）
- 禁止直接抛出 `RuntimeException` 或 `Exception`，必须使用有语义的异常类型

## 全局异常处理（NON-NEGOTIABLE）

- 使用 `@RestControllerAdvice` + `@ExceptionHandler` 统一捕获异常
- 全局处理器返回统一响应体（参考接口契约规范）
- 以下异常必须全局处理：
  - `BusinessException` 及其子类
  - `MethodArgumentNotValidException`（参数校验失败）
  - `HttpMessageNotReadableException`（请求体解析失败）
  - 兜底 `Exception`（避免泄露堆栈信息到响应中）

## 日志规范

- 使用 `SLF4J` + `Logback` 或 `Log4j2`，禁止直接使用 `System.out.println`
- 通过 `@Slf4j`（Lombok）或手动 `LoggerFactory.getLogger` 获取 Logger
- 生产环境禁止 DEBUG 日志；开发/测试环境可开启

### 日志级别使用场景

| 级别 | 适用场景 |
|------|----------|
| ERROR | 系统异常、不可预期错误、需立即处理的问题 |
| WARN | 可恢复的业务异常、配置缺失但有兜底 |
| INFO | 关键业务操作节点（订单创建、支付完成等） |
| DEBUG | 开发调试信息，生产禁止输出 |

### 结构化日志

- 关键操作日志包含：操作类型、业务 ID、操作人、耗时
- 推荐使用 MDC 注入 traceId / requestId，便于链路追踪

## 禁止事项

- 禁止在 catch 块中吞掉异常（即 catch 后不记录、不抛出）
- 禁止将异常堆栈直接返回给客户端
- 禁止在循环内大量打印日志（性能风险）
