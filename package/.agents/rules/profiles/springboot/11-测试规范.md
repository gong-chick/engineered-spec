# 测试规范（Spring Boot）

## 测试框架

- 统一使用 **JUnit 5**（`@ExtendWith(SpringExtension.class)` 或 `@SpringBootTest`）
- Mock 框架：**Mockito**（`@MockBean`、`@Mock`、`@InjectMocks`）
- 断言库：**AssertJ**（推荐）或 JUnit 5 原生 `Assertions`

## 测试分层策略

| 层次 | 注解 | 范围 | 说明 |
|------|------|------|------|
| 单元测试 | 无 Spring 上下文 | Service / 工具类 | 使用 Mockito 模拟依赖，运行最快 |
| 切片测试 | `@WebMvcTest`、`@DataJpaTest` | Controller / Repository | 只加载对应层的 Spring 上下文 |
| 集成测试 | `@SpringBootTest` | 全链路 | 加载完整上下文，速度慢，按需使用 |

## 测试文件约定

| 项目 | 约定 |
|------|------|
| 文件位置 | `src/test/java/<same-package-as-source>/` |
| 文件命名 | `XxxTest.java`（单元测试）、`XxxIT.java`（集成测试） |
| 测试配置 | `src/test/resources/application-test.yml` |

## 编写规范

- 测试方法命名使用中文或英文描述，表达预期行为而非实现步骤
- 遵循 **Arrange-Act-Assert** 模式
- 每个测试方法只验证一个行为点
- 测试应互相独立，不依赖执行顺序
- 使用 `@TestPropertySource` 或测试配置文件覆盖生产配置

## 质量门禁（NON-NEGOTIABLE）

- 所有 Service 层核心业务逻辑必须有单元测试覆盖
- `@SpringBootTest` 集成测试按需使用，不要求全覆盖
- 测试必须能独立运行，不依赖外部真实服务（使用 Mock 或嵌入式数据库）
- 新增工具类必须附带测试用例
