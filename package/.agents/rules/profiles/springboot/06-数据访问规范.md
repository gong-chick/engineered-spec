# 数据访问规范（Spring Boot）

## ORM 框架约定

- 项目统一使用团队约定的 ORM 框架（MyBatis / JPA / Spring Data），禁止混用
- MyBatis 项目：SQL 写在 XML 中（`src/main/resources/mapper/`），禁止直接在注解里写复杂 SQL
- JPA 项目：简单查询使用 Repository 方法命名，复杂查询使用 `@Query` 或 Specification

## Repository 命名

- 接口命名：`XxxMapper`（MyBatis）或 `XxxRepository`（JPA）
- 每个实体对应一个 Mapper / Repository，禁止跨实体混合操作

## 分页查询

- 统一使用框架提供的分页机制（MyBatis-Plus `Page`、Spring Data `Pageable`）
- 禁止手写 `LIMIT offset, size` 做内存分页

## SQL 安全

- 禁止字符串拼接构造 SQL（防 SQL 注入）
- 动态 SQL 使用框架提供的条件拼接（`<if>`、`Criteria`）
- 批量操作使用框架批量接口，禁止循环单条操作

## 实体设计

- 主键统一通过框架生成策略管理（雪花 ID、自增、UUID 按项目选一）
- 审计字段（创建时间、更新时间、创建人、更新人）统一由基类或 `@EntityListeners` 自动填充
- 逻辑删除字段统一命名（如 `is_deleted`），不做物理删除
