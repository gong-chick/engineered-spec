# 规则生成范围与写入清单

## 目标

在真正扫描项目之前，先确定本轮必须写入哪些文件，避免只做分析不落盘，或只写 `01/03` 就提前结束。

## 固定写入

以下产物每次 `project-init` 都要生成或刷新：

- `.agents/rules/01-项目概述.md`
- `.agents/rules/03-项目结构.md`
- `context/PROJECT.md`

## 条件写入

### OpenSpec 项目说明

- 若存在 `openspec/`，同步 `openspec/project.md`
- 若不存在 `openspec/`，跳过，不强制创建 OpenSpec

### 自定义规则

以下规则只在“缺失”或“manifest 要求刷新”时进入本轮写入清单：

| 编号 | 文件名 | 说明 |
|------|--------|------|
| 04 | `04-组件规范.md` | 组件结构、目录组织、编写约定 |
| 05 | `05-API规范.md` | 接口目录、请求封装、命名、错误处理 |
| 06 | `06-路由规范.md` | 路由配置、懒加载、守卫、目录结构 |
| 07 | `07-状态管理.md` | Store 目录、模块划分、编写规范 |
| 09 | `09-样式规范.md` | 样式方案、主题变量、全局样式 |

## 判断逻辑

### 第一步：看 manifest

检查 `.ai-spec/manifest.json`：

- 若存在 `local_preferences.project_init.custom_rules`
- 则其中列出的规则进入 `待刷新列表`

### 第二步：看规则文件是否缺失

检查 `.agents/rules/`：

- 若 `04/05/06/07/09` 中某项不存在
- 则该项进入 `待生成列表`

### 第三步：合并成本轮写入清单

- 若某条规则同时出现在 `待刷新列表` 和 `待生成列表`
- 按“生成或刷新”统一处理
- 若文件已存在且不在 `待刷新列表`
- 默认视为已落盘或已人工维护，不覆盖

## 本轮写入清单模板

执行前必须形成一份明确清单：

- 固定写入：
  - `.agents/rules/01-项目概述.md`
  - `.agents/rules/03-项目结构.md`
  - `context/PROJECT.md`
- 条件写入：
  - `openspec/project.md`（仅当 `openspec/` 存在）
  - `待生成列表` 中缺失的 `04/05/06/07/09`
  - `待刷新列表` 中需要按项目事实重写的 `04/05/06/07/09`

后续执行必须以这份清单为准，逐项落盘，不得遗漏。

## 约束

- 只要 `待生成列表` 非空，只写 `01/03 + PROJECT` 视为未完成
- 不允许把“后续再补 04/05/06/07/09”写进摘要后直接结束
- 不允许只分析、不写文件
