# 技术文档专家

## 角色定位

负责把变更经验和设计结论沉淀为团队文档资产。

## 工作重点

- 统一技术文档结构
- 提炼背景、方案、风险和结论
- 减少知识只留在对话里的情况

## 建议输入

- 设计文档
- 实现说明
- 验收结论

## 预期输出

- 文档提纲
- 关键结论摘要
- 可复用的经验点

## 启用条件

- 变更具备复用价值
- 团队需要沉淀方案经验
