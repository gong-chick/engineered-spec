这是 Cursor 的兼容入口，语义对齐 `/opsx:explore`。

目标：
- 围绕一个主题做探索、调研、架构讨论或范围澄清
- 可以读代码、查现状、画结构、比方案
- 不直接实现业务代码
- 当思路收敛时，自然过渡到 `/opsx-propose`

工作方式：

1. 先确认探索主题  
   - 若用户已给出主题，直接围绕它展开
   - 若主题很模糊，可先问一两句自然语言澄清

2. 允许做的事：
   - 读代码和文档
   - 搜索仓库
   - 对比方案与权衡
   - 用表格或 ASCII 图梳理结构
   - 引用已有 OpenSpec 变更作为上下文

3. 禁止做的事：
   - 不直接写业务实现代码
   - 不把探索直接变成实施
   - 不在未达成一致时强行创建产物

4. 当探索逐渐收敛时，可以主动提示：
   - 若准备正式立项：`/opsx-propose`
   - 若只是补充思路：继续探索即可

5. 输出风格：
   - 以讨论、比较、澄清为主
   - 保持结构清楚
   - 不使用旧版重型问答流程

守卫：
- 这是探索模式，不是实现模式
- Cursor 用 `/opsx-explore`；其他 IDE 仍走 `/opsx:explore`
