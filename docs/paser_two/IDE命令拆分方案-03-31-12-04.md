# IDE命令拆分方案

## 1. 结论

当前 `IDE（开发工具）` 真实触发入口已从一个统一命令，拆成两套更具体的模板：

- `spec-start（启动任务）`
- `spec-continue（继续任务）`

并保留：

- `spec-orchestrate（统一兼容入口）`

## 2. 为什么这样拆

目的不是增加命令数量，而是降低主代理误判：

- 新任务启动时，只允许产出 `task-orchestrator-bootstrap（主代理首轮桥接载荷）`
- 运行中继续推进时，只允许产出 `task-orchestrator-runtime-action（主代理运行动作载荷）`

这样：

- 提示词更短
- 主代理职责更清楚
- 后面插件页面更容易映射成两个按钮：
  - `开始任务`
  - `继续任务`

## 3. 当前命令建议

### 3.1 启动任务

```text
/spec-start 创建一个商品组件
```

### 3.2 继续任务

```text
/spec-continue 审批通过，继续
/spec-continue 当前任务取消
/spec-continue 查看当前状态
```

### 3.3 兼容入口

```text
/spec-orchestrate
```

适合：

- 还没切换旧习惯
- 需要统一入口兜底

## 4. 当前价值

拆分后，`IDE（开发工具）` 的真实入口已经更接近后续平台页面：

- `spec-start（启动任务）` 对应“开始任务”
- `spec-continue（继续任务）` 对应“继续任务”

这比只保留一个 `/spec-orchestrate` 更稳定，也更符合用户心智。
