# TASK_开发任务拆解 — BR AI Spec Visual 首页

> 📌 **适用对象**：前端开发工程师、后端开发工程师、测试工程师、项目经理  
> 📖 **关联文档**：[PRD](./PRD_产品需求文档.md) · [TECH](./TECH_技术架构文档.md) · [UI](./UI_设计规范文档.md) · [API](./API_接口定义文档.md)  
> 🎯 **核心目标**：将首页需求拆解为可执行、可验收的开发任务

---

## 1. 研发里程碑

| 里程碑 | 目标 | 交付项 | 验收标准 |
|--------|------|--------|----------|
| **M1：能接入** | 完成首页框架 + Onboarding 报告 | 首页布局、Onboarding 卡片、Demo Workspace | 首页可访问，Onboarding 状态正确显示 |
| **M2：能闭环** | 完成运行态 + 交付进度 + 效率收益 | 运行态健康度、交付闭环进度、效率收益卡 | 数据同步正常，首页可看见完整闭环 |
| **M3：能复盘** | 完成阻塞 + 资产 + 趋势 | 阻塞变化流、规范资产命中、安装趋势 | 有接入数据、归档数据、阻塞点分析 |
| **M4：能推广** | 完成试点专区 + 复盘 | 试点项目专区、试点复盘 | 有场景模板、有文档体系、有平台演示能力 |
| **M5：首版发布** | 全量测试 + 性能优化 + 文档完善 | 测试报告、性能报告、用户文档 | 通过 E2E 测试，首屏加载 < 2s |
