# UI 设计规范文档 — BR AI Spec Visual 首页

> 📌 **适用对象**：前端开发工程师、UI/UX 设计师、产品经理  
> 📖 **关联文档**：[PRD](./PRD_产品需求文档.md) · [TECH](./TECH_技术架构文档.md) · [API](./API_接口定义文档.md)  
> 🎯 **核心目标**：定义 BR AI Spec Visual 首页的视觉风格、模块布局、交互状态与组件规范

---

## 1. 设计目标与原则

### 1.1 首页定位

首页不是系统结构索引页，而是：

**规范驱动 AI开发的交付驾驶舱首页**

- 观测清晰度参考 **Grafana**
- 行动引导参考 **Cockpit**

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| 动作优先 | 第一屏先回答"是否接入成功、现在是否在产生价值"，而非按系统结构堆模块 |
| 信息分层 | 能力可以全，首页必须收敛。L1 必须展示 → L2 体现价值 → L3 可推广 → L4 扩展 |
| 空态有引导 | 不能只显示"暂无数据"，必须同时给出原因说明 + 下一步动作 |
| 加载有序 | 首页骨架屏，核心卡片优先加载，趋势图与列表可后置加载 |
| 异常可恢复 | Collector 断连、Workspace 无权限等异常状态需有明确提示和修复入口 |

## 2. 首页信息架构（4 层结构）

### L1：必须优先展示

| 模块 | 说明 |
|------|------|
| Onboarding 报告 | 解决首次接入用户"不知道从哪里开始、不知道接入是否成功、不知道下一步做什么"的问题 |
| 运行态健康度 | 快速让用户知道当前系统是不是在"正常跑" |
| 交付闭环进度 | 体现规范驱动开发是否真正产生"从发起到归档"的结果 |
| 效率收益卡 | 体现这套系统值不值得继续使用 |

### L2：体现平台价值

| 模块 | 说明 |
|------|------|
| 阻塞变化流 | 让用户看到当前需求为什么没有继续往前推进 |
| 规范资产命中情况 | 让用户知道项目是否真的完成规范接入，而不是只装了一个壳 |
| 最近 Runs | 展示当前运行中的需求列表 |

### L3：体现可推广能力

| 模块 | 说明 |
|------|------|
| 最近 Changes / Specs | 展示最近变更与规范文档 |

| 试点项目专区 | 当前试点项目列表、状态、归档完成率 |

### L4：扩展模块（首版不实现）

| 模块 | 说明 |
|------|------|
| 试点复盘与建议动作 | 试点总结与下一步建议 |
| 规范资产完整度评分 | 规则、技能、OpenSpec、.ai-spec 完整度评分 |

## 4. 10 个模块详细设计

### 模块 A：Onboarding 报告

#### 模块目标

解决首次接入用户"不知道从哪里开始、不知道接入是否成功、不知道下一步做什么"的问题。

#### 展示内容

**当前状态阶段（进度条）：**

| 阶段 | 图标 | 说明 |
|------|------|------|
| 未接入 | 🔴 | 尚未安装 ai-spec-auto |
| 已安装 Base | 🟡 | 已执行 `npx @engineered/ai-spec-auto init .` |
| 已联通 Visual | 🟢 | Collector 上报成功 |
| 已启动首个需求 | 🔵 | 首个 Run 已创建 |
| 已完成首个归档 | 🟣 | 首个 Change 已归档 |

**资产检测结果（清单式）：**

| 资产 | 状态 | 缺失提示 |
|------|------|----------|
| rules | ✅/❌ | 请检查 .agents/rules 目录是否存在 |
| skills | ✅/❌ | 请检查 .agents/skills 目录是否存在 |
| OpenSpec | ✅/❌ | 请检查 openspec 目录是否存在 |
| .ai-spec | ✅/❌ | 请检查 .ai-spec 目录是否存在 |
| registry | ✅/❌ | 请检查 .agents/registry 目录是否存在 |
| logs | ✅/❌ | 请检查 .omx/logs 目录是否存在 |

**下一步建议动作：**

| 当前状态 | 建议动作 |
|----------|----------|
| 未接入 | 执行 `npx @engineered/ai-spec-auto init .` 开始接入 |
| 已安装 Base | 运行 Collector 联通 Visual |
| 已联通 Visual | 开始一个需求，跑通首个闭环 |
| 已启动首个需求 | 等待归档完成，查看首页数据 |
| 已完成首个归档 | 查看效率收益，开始第二个需求 |

**快速入口（按钮组）：**

- [开始一个需求] → 跳转 Runs 页面
- [打开 Demo Workspace] → 跳转 Demo Workspace
- [查看 10 分钟快速开始] → 跳转文档
- [查看命令速查] → 跳转命令速查页

#### 交互要求

- 支持根据当前接入状态动态变化
- 支持点击后跳转到对应引导或页面
- 若检测缺失资产，需给出明确提示与修复入口

#### 空态文案

```
📦 尚未接入任何项目

接入 ai-spec-auto 后，首页将显示：
• 接入状态与资产检测结果
• 运行态健康度与交付进度
• 效率收益与规范资产命中情况

[开始接入] [查看 10 分钟快速开始]
```

## 5. 页面状态设计

### 5.1 空状态

**设计原则：**

- 不能只显示"暂无数据"
- 必须同时给出原因说明 + 下一步动作
- 提供快捷入口按钮

**空态模板：**

```
[图标] [标题]

[原因说明]

接入 [产品] 后，首页将显示：
• [展示内容 1]
• [展示内容 2]
• [展示内容 3]

[行动按钮 1] [行动按钮 2]
```

### 5.2 加载状态

**设计原则：**

- 首页骨架屏
- 核心卡片优先加载
- 趋势图与列表可后置加载

**骨架屏组件：**

| 模块 | 骨架屏类型 |
|------|-----------|
| Onboarding | 卡片骨架（大） |
| 运行态健康度 | 数字骨架 + 折线图骨架 |
| 交付闭环进度 | 数字骨架 + 进度条骨架 |
| 效率收益卡 | 数字骨架 + 柱状图骨架 |
| 阻塞变化流 | 条形图骨架 |
| 规范资产命中 | 清单骨架 |
| 最近 Runs | 表格骨架 |
| 最近 Changes | 表格骨架 |
| 安装趋势 | 折线图骨架 |
| 试点项目 | 卡片骨架 |

### 5.3 异常状态

| 异常类型 | 提示文案 | 修复入口 |
|----------|----------|----------|
| Collector 断连 | Collector 未上报数据，请检查 Collector 配置 | [查看 Collector使用指南] |
| Workspace 无权限 | 当前用户无权限访问此 Workspace | [联系管理员] |
| 统计数据暂不可用 | 统计数据正在同步中，请稍后刷新 | [刷新] |
| 请求参数错误 | 请求参数有误，请刷新页面重试 | [刷新] |

## 11. 设计规范总结

| 设计维度 | 规范 |
|----------|------|
| 布局方式 | 12 栅格 + 响应式 |
| 间距规范 | 24px 边距 / 16px 模块间距 / 12px 卡片间距 / 16px 内边距 |
| 空态原则 | 原因说明 + 下一步动作 + 快捷入口 |
| 加载原则 | 骨架屏 + 核心优先 + 趋势后置 |
| 异常原则 | 明确提示 + 修复入口 |
| 文案原则 | 动作优先 + 信息分层 + 术语统一 |
| 埋点原则 | 曝光 + 点击 + 交互全覆盖 |
