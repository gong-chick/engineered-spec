#### 解决方案 C：服务器部署

**详见**：`docs/five/发版与部署指南.md`

**核心流程**：

```bash
# 1. 发布新版本（如需要）
cd /Users/lizhenwei/workspace/vueworkspace/bairong/br-ai-spec
npm version minor  # 2.0.0 -> 2.1.0
git push origin main --tags

# 2. 部署 visual 服务到内网服务器
ssh user@10.0.1.100
git clone https://github.com/Colouful/engineered-spec-visual.git
cd engineered-spec-visual
cp .env.example .env
vim .env  # 设置 DB_PASSWORD 和 NEXTAUTH_SECRET
docker-compose up -d

# 3. 在项目中配置 visual
cd /path/to/project
cat > .ai-spec/visual-config.json <<EOF
{
  "enabled": true,
  "visual_url": "http://10.0.1.100:3000",
  "workspace_id": "project-name"
}
EOF

# 4. 执行 Collector 采集历史数据
cd /path/to/engineered-spec-visual
npm run collector -- \
  --workspace-id project-name \
  --project /path/to/project \
  --server http://10.0.1.100:3000
```

### 问题 2：隐藏 Init 提示并默认开启

#### 解决方案：修改 install-workflow.js

**修改内容**：

```diff
# File: bin/install-workflow.js

# Line 170: 修改默认值
- visualBridge: 'ask',
+ visualBridge: 'yes',  // 默认启用，不再提示

# Line 1157-1162: 简化交互逻辑
  if (options.visualBridge === 'ask') {
-   console.log('');
-   info('是否启用 Visual 平台桥接？');
-   console.log('  启用后会在 .ai-spec/visual-bridge.json 落可视化桥接配置，但默认不影响现有协议执行链。');
-   options.visualBridge = (await confirm('启用 visual bridge?', false)) ? 'yes' : 'no';
-   ok(options.visualBridge === 'yes' ? '将启用 visual 平台桥接配置' : '跳过 visual 平台桥接配置');
+   // 默认启用 visual bridge，不再提示用户
+   options.visualBridge = 'yes';
  }
```

## 常见问题

### Q1：测试项目更新后，visual-hooks 找不到？

**A**：清除缓存重新安装

```bash
cd /path/to/test/project
rm -rf node_modules/ai-spec-auto
npm install /Users/lizhenwei/workspace/vueworkspace/bairong/br-ai-spec
```

### Q2：Init 命令仍然显示提示？

**A**：验证代码是否已更新

```bash
# 检查代码
grep -n "visualBridge: 'yes'" /Users/lizhenwei/workspace/vueworkspace/bairong/br-ai-spec/bin/install-workflow.js

# 应该在 Line 170 看到 visualBridge: 'yes'
# 如果看到 visualBridge: 'ask'，说明代码未更新
```

### Q3：如何在多个项目中批量更新？

**A**：编写批量脚本

```bash
#!/bin/bash
AUTO_PATH="/Users/lizhenwei/workspace/vueworkspace/bairong/br-ai-spec"

PROJECTS=(
  "/Users/lizhenwei/workspace/test/test-ai-spec/prd-to-delivery-local-first-060/test_副本10"
  "/path/to/another/project"
)

for project in "${PROJECTS[@]}"; do
  echo "Updating $project..."
  cd "$project"
  npm install "$AUTO_PATH"
  echo "✅ $project updated"
done
```

### Q4：如何禁用 visual 集成？

**A**：方法 1：删除配置

```bash
rm .ai-spec/visual-config.json
```

**方法 2**：修改配置

```bash
echo '{"enabled": false}' > .ai-spec/visual-config.json
```
