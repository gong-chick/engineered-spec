## 验证流程

### 步骤 1：准备 visual 项目

#### 1.1 安装依赖

```bash
cd /Users/lizhenwei/workspace/vueworkspace/bairong/engineered-spec-visual
npm install
```

#### 1.2 配置环境变量（使用 SQLite）

创建 `.env` 文件：

```bash
cat > .env <<EOF
# Database (使用 SQLite 进行本地开发)
DATABASE_URL="file:./dev.db"

# NextAuth
NEXTAUTH_SECRET="$(openssl rand -base64 32)"
NEXTAUTH_URL="http://localhost:3000"

# 本地开发模式
NODE_ENV=development
EOF
```

#### 1.3 初始化数据库

```bash
# 生成 Prisma 客户端
npm run prisma:generate

# 初始化数据库
npm run prisma:push
```

**预期输出**：

```
✔ Generated Prisma Client
✔ The database is now in sync with the Prisma schema.
```

#### 1.4 启动开发服务器

**在新终端窗口运行**：

```bash
cd /Users/lizhenwei/workspace/vueworkspace/bairong/engineered-spec-visual
npm run dev
```

**预期输出**：

```
▲ Next.js 16.2.4
- Local:        http://localhost:3000
- Environments: .env

✓ Ready in 2.3s
```

**验证**：

```bash
# 在另一个终端验证
curl http://localhost:3000/api/health

# 预期输出：{"ok":true}
```

或在浏览器打开：`http://localhost:3000`

### 步骤 3：配置 visual 集成


## 常见问题

### Q1：visual 服务启动失败

**错误**：`Error: Cannot find module 'prisma'`

**解决**：

```bash
cd /Users/lizhenwei/workspace/vueworkspace/bairong/engineered-spec-visual
npm install
npm run prisma:generate
```

### Q2：数据库初始化失败

**错误**：`Error: P1003: Database does not exist`

**解决**：

```bash
# 确保 .env 中使用 SQLite
cat .env | grep DATABASE_URL

# 应该看到：DATABASE_URL="file:./dev.db"

# 重新初始化
npm run prisma:push
```

### Q3：Collector 报错 "ECONNREFUSED"

**错误**：`Error: connect ECONNREFUSED 127.0.0.1:3000`

**原因**：visual 服务未启动

**解决**：

```bash
# 检查 visual 服务是否运行
curl http://localhost:3000/api/health

# 如果失败，重新启动
cd /Users/lizhenwei/workspace/vueworkspace/bairong/engineered-spec-visual
npm run dev
```

### Q4：协议推进时看不到 visual-hooks 日志

**原因 1**：配置未启用

```bash
# 检查配置
cat .ai-spec/visual-config.json

# 确保 "enabled": true
```

**原因 2**：visual-hooks 未安装

```bash
# 检查 visual-hooks
ls node_modules/ai-spec-auto/internal/visual-hooks/

# 如果不存在，重新安装
npm install /Users/lizhenwei/workspace/vueworkspace/bairong/br-ai-spec
```

### Q5：visual 控制台看不到数据

**可能原因**：

1. **workspace_id 不匹配**
   ```bash
   # 检查配置中的 workspace_id
   cat .ai-spec/visual-config.json | grep workspace_id
   
   # 在 visual 控制台筛选对应的 workspace
   ```

2. **Collector 未执行或失败**
   ```bash
   # 重新执行 Collector
   cd /Users/lizhenwei/workspace/vueworkspace/bairong/engineered-spec-visual
   npm run collector -- \
     --workspace-id test-project-local \
     --project /Users/lizhenwei/workspace/test/test-ai-spec/prd-to-delivery-local-first-060/test_副本10 \
     --server http://localhost:3000
   ```

3. **实时推送失败**
   ```bash
   # 检查 visual 服务日志
   # 应该看到 HTTP POST 请求日志
   ```
