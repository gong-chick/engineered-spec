### 方式 2：正式发版（推荐用于生产项目）

**适用场景**：发布稳定版本到 npm 或内部私有仓库

#### 步骤 1：更新版本号

```bash
cd /Users/lizhenwei/workspace/vueworkspace/bairong/br-ai-spec

# 查看当前版本
cat package.json | grep version

# 更新版本号
npm version patch   # 小版本：2.0.0 -> 2.0.1
# 或
npm version minor   # 中版本：2.0.0 -> 2.1.0
# 或
npm version major   # 大版本：2.0.0 -> 3.0.0

# 这会自动创建 git tag 并提交
```

#### 步骤 2：发布到 npm（如果是公开仓库）

```bash
# 登录 npm（首次需要）
npm login

# 发布
npm publish
```

#### 步骤 3：发布到内部私有仓库（推荐）

**方案 A：使用 Git 仓库作为依赖源**

```bash
# 1. 推送代码和 tag
git push origin main
git push origin --tags

# 2. 在项目中使用 Git 依赖
cd /path/to/your/project
npm install git+http://git.internal/br-ai-spec.git#v2.1.0
```

**方案 B：使用 Verdaccio 私有 npm 仓库**

```bash
# 1. 配置私有仓库
npm config set registry http://npm.internal:4873

# 2. 发布
npm publish

# 3. 在项目中安装
cd /path/to/your/project
npm install ai-spec-auto@latest
```

**方案 C：使用文件系统共享**

```bash
# 1. 打包
cd /Users/lizhenwei/workspace/vueworkspace/bairong/br-ai-spec
npm pack
# 生成：ai-spec-auto-2.1.0.tgz

# 2. 复制到共享目录
cp ai-spec-auto-2.1.0.tgz /mnt/shared/packages/

# 3. 在项目中安装
cd /path/to/your/project
npm install /mnt/shared/packages/ai-spec-auto-2.1.0.tgz
```

```bash
# 4. 验证
curl http://10.0.1.100:3000/api/health
```

### 场景 2：更新所有项目的 visual 配置

**批量配置脚本**：

```bash
#!/bin/bash
# update-visual-config.sh

VISUAL_SERVER="http://10.0.1.100:3000"

PROJECTS=(
  "/path/to/project-a:project-a"
  "/path/to/project-b:project-b"
  "/Users/lizhenwei/workspace/test/test-ai-spec/prd-to-delivery-local-first-060/test_副本10:test-project-local"
)

for item in "${PROJECTS[@]}"; do
  IFS=':' read -r path workspace_id <<< "$item"
  
  echo "Configuring $workspace_id at $path..."
  
  cat > "$path/.ai-spec/visual-config.json" <<EOF
{
  "enabled": true,
  "visual_url": "$VISUAL_SERVER",
  "workspace_id": "$workspace_id",
  "workspace_name": "$workspace_id",
  "push_mode": "hook",
  "push_timeout_ms": 3000,
  "retry_times": 1
}
EOF

  echo "✅ $workspace_id configured"
done

echo ""
echo "🎉 All projects configured!"
```

执行：

```bash
chmod +x update-visual-config.sh
./update-visual-config.sh
```

### 场景 3：执行批量 Collector 采集

```bash
#!/bin/bash
# collect-all-projects.sh

VISUAL_SERVER="http://10.0.1.100:3000"
COLLECTOR_CLI="/Users/lizhenwei/workspace/vueworkspace/bairong/engineered-spec-visual"

PROJECTS=(
  "/path/to/project-a:project-a"
  "/path/to/project-b:project-b"
  "/Users/lizhenwei/workspace/test/test-ai-spec/prd-to-delivery-local-first-060/test_副本10:test-project-local"
)

for item in "${PROJECTS[@]}"; do
  IFS=':' read -r path workspace_id <<< "$item"
  
  echo "Collecting $workspace_id from $path..."
  
  cd "$COLLECTOR_CLI"
  npm run collector -- \
    --workspace-id "$workspace_id" \
    --project "$path" \
    --server "$VISUAL_SERVER" \
    --json
  
  echo "✅ $workspace_id collected"
  echo ""
done

echo "🎉 All projects collected!"
```

执行：

```bash
chmod +x collect-all-projects.sh
./collect-all-projects.sh
```

## 更新流程总结

### 对于测试项目（test_副本10）

**一键更新脚本**：

```bash
#!/bin/bash
# update-test-project.sh

AUTO_PATH="/Users/lizhenwei/workspace/vueworkspace/bairong/br-ai-spec"
TEST_PROJECT="/Users/lizhenwei/workspace/test/test-ai-spec/prd-to-delivery-local-first-060/test_副本10"

echo "=== 更新测试项目 ==="

# 1. 更新 ai-spec-auto
echo "1. 更新 ai-spec-auto..."
cd "$TEST_PROJECT"
npm install "$AUTO_PATH"

# 2. 验证更新
echo "2. 验证更新..."
VERSION=$(npx ai-spec-auto --version)
echo "   当前版本: $VERSION"

# 3. 检查 visual-hooks
if [ -d "node_modules/ai-spec-auto/internal/visual-hooks" ]; then
  echo "   ✅ visual-hooks 已安装"
else
  echo "   ❌ visual-hooks 未找到"
  exit 1
fi

# 4. 创建 visual 配置（如果不存在）
if [ ! -f ".ai-spec/visual-config.json" ]; then
  echo "3. 创建 visual 配置..."
  cat > .ai-spec/visual-config.json <<EOF
{
  "enabled": true,
  "visual_url": "http://localhost:3000",
  "workspace_id": "test-project-local",
  "workspace_name": "测试项目本地实例"
}
EOF
  echo "   ✅ 配置文件已创建"
else
  echo "3. visual 配置已存在，跳过"
fi

# 5. 测试协议命令
echo "4. 测试协议命令..."
npx ai-spec-auto protocol-status > /dev/null 2>&1
if [ $? -eq 0 ]; then
  echo "   ✅ 协议命令正常"
else
  echo "   ⚠️  协议命令异常，请检查"
fi

echo ""
echo "=== 更新完成 ==="
```

保存为 `update-test-project.sh` 并执行：

```bash
chmod +x update-test-project.sh
./update-test-project.sh
```

## 常见问题

### Q1：测试项目中 npm link 后，visual-hooks 找不到？

**A**：npm link 有时会丢失符号链接。解决方案：

```bash

# 方案 2：直接安装本地路径
```

**A**：清除缓存重新安装：


### Q3：如何在多个测试项目中同步更新？

**A**：使用批量更新脚本：

```bash
#!/bin/bash
AUTO_PATH="/Users/lizhenwei/workspace/vueworkspace/bairong/br-ai-spec"

  "/Users/lizhenwei/workspace/test/test-ai-spec/prd-to-delivery-local-first-060/test_副本10"
```

## 版本管理建议

### 版本号规范

遵循 [Semantic Versioning](https://semver.org/)：

- **主版本号（Major）**：不兼容的 API 变更（如 2.0.0 -> 3.0.0）
- **次版本号（Minor）**：向后兼容的功能新增（如 2.0.0 -> 2.1.0）
- **修订号（Patch）**：向后兼容的问题修复（如 2.0.0 -> 2.0.1）

### 本次集成的版本建议

**推荐版本**：`2.1.0`（次版本号升级）

**理由**：
- 新增 visual hooks 功能（新特性）
- 向后兼容（不影响现有项目）
- 默认不启用（需显式配置）
