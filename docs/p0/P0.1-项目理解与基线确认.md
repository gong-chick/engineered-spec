## 4. 核心模块分析

### 4.1 初始化流程（init）

```
cli.js → init-command.js
  → InitService.createPlan()
    → TechScannerEngine.scan()          # 扫描项目技术栈
    → InitPlanBuilder.build()           # 构建初始化计划
    → ManifestInstaller.recommendForPackage()  # 推荐 Manifest
  → InitApplier.apply()
    → ManifestInstaller.install()       # 安装 .agents 资产
    → IdeLinker.link()                  # 创建 .cursor/.claude 符号链接
    → ProjectConfigWriter.write()       # 写入 .ai-spec/project.json
    → PolicyConfigWriter.write()        # 写入 .ai-spec/policy.json
    → WorkspaceConfigWriter.write()     # 写入 .ai-spec/workspace.json
    → LockFileWriter.write()            # 写入 .ai-spec/ai-spec.lock.json
    → RegistryIndexWriter.write()       # 写入 .agents/registry.index.json
    → ContextIndexWriter.write()        # 写入 .ai-spec/context-index.json
    → IdePointerInjector.write()        # 写入 CLAUDE.md/.cursor/rules/ai-spec-auto.mdc 等指针文件
```

### 4.2 技术栈扫描（scanner）

支持检测的框架/技术栈：
- React (Vite/Webpack/Next.js)
- Vue (Vite)
- Spring Boot/MVC/Cloud
- NestJS
- FastAPI
- Go

### 4.3 IDE 适配（ide）

- **CursorAdapter**：生成 `.cursor/rules/ai-spec-auto.mdc` 和 `.cursor/commands/*.md`
- **ClaudeAdapter**：生成 `.claude/ai-spec-auto.md` 和 `.claude/commands/*.md`
- **IdeLinker**：创建 `.cursor/rules → .agents/rules` 和 `.claude/rules → .agents/rules` 符号链接
- **IdePointerInjector**：在 CLAUDE.md、memory.md 等文件中注入管理区块

### 4.4 检查服务（check）

- `CheckService.check()` → `AssetTamperChecker.check()`
- 检查资产完整性、lock/registry/context-index 一致性和隐私配置
