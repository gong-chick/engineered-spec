# V1.1 FAQ

## 1. Visual 页面 URL 可以作为 --visual-url 吗？

不可以。`--visual-url` 必须使用 API Base(API 基地址)：

```text
http://172.20.10.4:18780
```

页面地址只用于浏览器查看：

```text
http://172.20.10.4:18780/w/test_%E5%89%AF%E6%9C%AC17
```

## 2. 真实业务项目能执行 init --yes 吗？

不能。V1.1 只允许对真实业务项目执行：

```bash
scan
scan --json
init --recommend --dry-run
```

`init --yes` 只能用于 `/tmp/ai-spec-v1-1-smoke` 这类临时项目。

## 3. Hub 不可用怎么办？

`init --recommend` 可以按配置 fallbackToLocal(降级本地推荐)。但 `sync` 如果缓存缺失，需要 Hub 可用。

## 4. Visual 不可用会阻断本地流程吗？

不会。Visual 上报是 non-blocking(非阻断)链路，失败会输出 warning。

## 5. 为什么 registry.index.json 不能保存 content？

`registry.index.json` 只保存索引和 checksum。资产正文只进入全局缓存：

```text
~/.ai-spec-auto/cache/assets/<checksum>/content.md
~/.ai-spec-auto/cache/agent-profiles/<checksum>/content.json
```

## 6. 哪些数据不能上报？

禁止上报源码、fileContent、rawPrompt、rawResponse、绝对路径、`/Users/` 路径、apiKey、password、token、secret、`.env` 内容。

## 7. spec-continue --execute --dry-run 会调用真实 AI 工具吗？

不会。dry-run 只验证 Executor Provider(执行器提供者)选择和 prepare(准备任务文件)，不调用 Codex / Cursor / Claude Code 真实执行。

## 8. 页面跳转登录页算服务不可用吗？

不算。页面返回 307 到登录页说明 Web 服务在线；Collector API 健康检查通过即可认为 Visual API 可联通。
