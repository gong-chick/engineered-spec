推荐阅读顺序：

1. 想看项目背景和规划，先看“AI规范开发平台背景、竞品与推广规划”
2. 想直接发团队、并且控制在 3-5 分钟可读完，先看“团队同学AI规范开发最佳实践-短文版”
3. 想看目录、关键文件、`/spec-start` 到归档的完整调用链，先看“开发人员规范化开发实践：为什么值得学，怎么开始用”
4. 想直接发团队、并且更偏实操落地，再看“团队同学AI规范开发最佳实践-从今天怎么开始到团队怎么放大”
5. 想直接发团队统一认知，再看“团队同学最佳实践-语雀发布版”
6. 想从团队使用视角理解价值和落地，再看“团队同学最佳实践-从背景到落地”
7. 团队新人再看“团队同学AI规范开发最佳实践”
8. 再看“5 分钟上手”建立命令和目录认知
9. 想了解全链路，再看第四阶段的“项目介绍与运行机制说明”
10. 想直接看代码层的底层实现和调用链，再看“当前项目底层实现、架构设计与执行调用流程说明”
11. 想复用“发版后安装验证 + smoke + auto-fix”脚本，再看“发版后验证脚本与测试快照说明”
12. 需要维护 `config.yaml` 和规范协作，再看“OpenSpec配置与协作深入版”
