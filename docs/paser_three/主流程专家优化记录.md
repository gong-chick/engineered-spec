# 主流程专家优化记录

## 背景

当前协议主链已经稳定为：

```text
task-orchestrator
  -> requirement-analyst
  -> frontend-implementer
  -> code-guardian
  -> before-archive
  -> archive-change（批准后）
```

这一轮优化不调整主链顺序，也不新增命令、gate 或 schema，而是把 4 个主流程专家真正收口成：

- `role` 负责阶段目标、边界、交接
- `rule` 负责项目硬约束
- `skill` 负责具体执行方法

## 当前主链基线

### requirement-analyst

- 阶段目标：把需求收敛成 `proposal.md / specs/ / design.md / tasks.md`
- 核心 rules：`project-overview / project-structure / api-standard / route-standard / style-standard`
- 核心 skills：`create-proposal / design-analysis`
- 必备输出：
  - `proposal.md`
  - `specs/`
  - `design.md`
  - `tasks.md`

### frontend-implementer

- 阶段目标：基于 proposal/specs/design/tasks 完成最小必要实现
- 核心 rules：`project-structure / component-standard / route-standard / api-standard / store-standard / style-standard`
- 核心 skills：
  - 页面任务：`create-view -> create-route -> theme-variables`
  - 组件任务：`create-component -> theme-variables`
  - 接口任务：`create-api`
  - 状态任务：`create-store`
  - 混合任务：`execute-task`
- 必备输出：
  - `code`
  - `implementation-notes`

### code-guardian

- 阶段目标：把 rules 翻译成 checklist，给出交付放行或阻断结论
- 核心 rules：`api-standard / route-standard / style-standard / test-standard / audit-report-standard`
- 核心 skills：
  - 页面/UI：`ui-verification`
  - 体验与交互规范：`web-design-guidelines`
  - 测试缺口：`create-test`
- 必备输出：
  - `checklist.md`
  - `iterations.md`

### archive-change

- 阶段目标：先做 `archive_preflight`，再合并规范并完成归档
- 核心 rules：`audit-report-standard`
- 核心 skills：`archive-change`
- 必备输出：
  - `openspec/specs/`
  - `openspec/changes/archive/`
  - 归档摘要

## 可选专家插入条件

### requirement-analyst 阶段

- `design-collaborator`
  - 适用：设计稿、视觉还原、像素级交互、复杂 UI 约束
- `api-contract-specialist`
  - 适用：接口字段、联调、mock/真实接口边界不清、契约不稳定

### frontend-implementer 阶段

- `unit-test-specialist`
  - 适用：store、复杂逻辑、高回归风险改动
- `performance-auditor`
  - 适用：大列表、首屏卡顿、性能指标退化、明确性能目标

### code-guardian 阶段

- `verification-reviewer`
  - 适用：验收口径不完整、需要多人确认、证据不足
- `unit-test-specialist`
  - 适用：工具函数、store、复杂逻辑存在补测缺口
- `performance-auditor`
  - 适用：守护阶段发现明确性能风险

## 本轮改动摘要

- 为 4 个主流程专家新增统一 guidance：
  - `artifact_contract`
  - `skill_selection_policy`
  - `handoff_checklist`
  - `optional_role_triggers`
- 为 `archive-change` 新增 `archive_preflight`
- 强化主专家正文，明确“先 rule、后 skill、再交接”的执行顺序
- 将下列可选专家状态从 `planned` 收口为 `active`，但保持 optional：
  - `design-collaborator`
  - `api-contract-specialist`
  - `unit-test-specialist`
  - `verification-reviewer`
  - `performance-auditor`
- 在 registry 中显式补齐 `prd-to-delivery.optional_roles`
- 补充文档入口，保证主流程专家增强可以持续追踪

## 后续待办

- 继续把 `code-guardian` 的阻断项与具体规则条目做更细粒度映射
- 视使用频率评估是否为主流程专家补更细分的文档型可选专家
- 在更多 runtime 场景下验证 API-only、store-only、archive-preflight 的稳定性
