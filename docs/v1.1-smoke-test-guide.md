# V1.1 Smoke Test Guide

## 目标

验证 V1.1 从契约测试升级到真实本地服务可跑、临时项目可初始化、真实项目可 dry-run 回归。

## 健康检查

Hub:

```bash
curl -s http://localhost:3000/api/hub/manifests/recommend \
  -H "Content-Type: application/json" \
  -d '{"workspace":{},"projectFacts":[]}'
```

Visual Collector:

```bash
curl -s http://172.20.10.4:18780/api/collector/project-state \
  -H "Content-Type: application/json" \
  -d '{"eventId":"health","projectId":"health","type":"single","techProfile":{},"manifest":{},"packages":[],"privacy":{"sourceCodeIncluded":false,"rawPromptIncluded":false,"rawResponseIncluded":false,"absolutePathIncluded":false},"reportedAt":"2026-04-26T00:00:00.000Z"}'

```

页面可能跳转登录页，Collector API 可用即可继续 smoke。

## 临时项目

只写 `/tmp/ai-spec-v1-1-smoke`，不安装依赖，不运行 `npm install`。

期望结果：

1. `scan` 识别 `vue-vite`。
2. `init --dry-run` 推荐来源为 Hub。
3. `init --yes` 写入 `.ai-spec`、`.agents` 和 IDE 指针文件。
4. `sync` 拉取 Hub Manifest / Asset / Agent Profile。
5. `check` 和 `guard assets` 通过。
6. `spec-start --dry-run` 创建 run，但不创建真实业务变更。
7. `spec-continue --execute --dry-run` 不调用真实外部执行器。

## 真实项目

真实业务项目和普通测试项目只允许 dry-run：

```bash
scan
scan --json
init --recommend --dry-run
```

执行前后检查 git status，确认没有新增 `.ai-spec` 或 `.agents`。

## 失败处理

1. Hub 404：确认服务端是否运行在 webpack dev mode(webpack 开发模式)。
2. checksum 不一致：不要放宽校验，先确认 Hub seed 数据。
3. Visual 上报失败：本地流程不阻断，但必须记录 warning。
4. 真实项目被误判：只修 Scanner / Detector / InitPlan，不写真实项目。
