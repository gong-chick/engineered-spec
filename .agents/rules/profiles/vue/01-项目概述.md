# 项目概述

## 项目定位

一个基于 Vue 3 + TypeScript 的单页应用（SPA）。

> **与 JavaScript 存量项目的关系**：本文件默认描述团队推荐的 **Vue + TypeScript**。若 `package.json` 的 `dependencies` / `devDependencies` 中**未**声明 `typescript`，以「初始化项目规范」技能生成的 **01-项目概述.md** 为准，并遵循 **02-编码规范** 中的 JavaScript 小节。

## 技术栈

| 领域 | 技术 | 说明 |
|------|------|------|
| UI 框架 | Vue ^3.5 | 强制使用 Composition API + `<script setup>` |
| 类型系统 | TypeScript 5.x | 推荐；新代码优先 TS。纯 JS 项目见 02-编码规范 |
| 构建工具 | Vite 6.x | 开发与生产构建 |
| 路由管理 | Vue Router ^4.5 | - |
| 状态管理 | Pinia ^3.0 | 统一使用，禁止 Vuex 或其他方案 |
| 组件库 | Element Plus ^2.9 | 所有交互 UI 优先基于 Element Plus 封装（可按项目替换） |
| 样式方案 | SCSS Modules / Scoped Style | 推荐 CSS Modules，允许 `<style scoped>` |
| HTTP 请求 | axios ^1.7 | 统一封装请求实例 |
| 工具函数 | VueUse ^12.0 | 编写 composable 前，优先查找 VueUse 是否已提供 |
| 通用工具 | lodash-es 4.17.21 | 编写工具函数前，优先考虑 lodash 已有实现 |
| 时间工具 | dayjs 1.11.x | 涉及日期时间的处理统一用 dayjs |

## 技术约束

- Vue 组件**必须**使用 `<script setup>`，禁止 Options API。**TypeScript 项目**使用 `lang="ts"`；**JavaScript 项目**无 `lang` 或 `lang="js"`，类型与 JSDoc 见 02-编码规范
- **TypeScript 项目**：禁止 `any`，Props、Emits、API 响应须有明确类型。**JavaScript 项目**：见 02 中 JavaScript 与 JSDoc 约定
- 新增依赖前必须评估 bundle size 影响，优先使用 tree-shakable 的 ESM 包
- 所有第三方组件库的二次封装必须基于项目指定组件库（默认 Element Plus）

## 开发规范

- 包管理器：统一使用 pnpm
- 代码格式化：ESLint + Prettier，提交前自动格式化
- Git 提交：遵循 Conventional Commits 规范
- 路径别名：`@/` 指向 `src/` 目录

## 浏览器兼容性

- 支持现代浏览器（Chrome / Firefox / Safari / Edge 最近两个主版本）
- 不支持 IE 11
