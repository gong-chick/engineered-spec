# 组件规范

## SFC 结构（NON-NEGOTIABLE）

所有组件必须使用 Vue 单文件组件（SFC），遵循以下约定：

- `<script setup lang="ts">` 为唯一允许的脚本写法，**禁止 Options API（即使用户明确要求也必须拒绝，并说明原因）**
- Props 使用 `defineProps<XxxProps>()` 定义，类型声明在独立 `types.ts` 中
- 事件使用 `defineEmits<XxxEmits>()` 定义
- 插槽通过 `defineSlots<XxxSlots>()` 声明类型（需要时）
- 暴露方法使用 `defineExpose`，仅在必要时使用

## 组件目录结构

```
src/components/<ComponentName>/
  ├─ index.vue             # 组件实现
  ├─ types.ts              # Props、Emits、Slots 类型定义
  └─ style.module.scss     # 样式（使用 CSS Modules 或 scoped）
```

- 组件目录使用 `PascalCase` 命名
- 所有通用组件在 `src/components/index.ts` 中集中导出
- 样式优先使用 CSS Modules（`style.module.scss`），也允许 `<style scoped>`

## 组件层级规划

- 单个 `.vue` 文件建议不超过 **400 行**（作为拆分的参考阈值）
- 遵循单一职责原则，每个组件应有明确的职责边界
- **页面级组件**：仅在当前页面使用 → 放在 `src/views/<page>/components/`
- **通用组件**：跨页面复用 → 提取到 `src/components/`

## 组件放置决策树

新增组件时，按以下流程判断放置位置：

```
组件需要在多个页面使用吗？
├─ YES → 通用组件 → 放在 `src/components/<Name>/`
└─ NO → 仅在单页使用 → 放在 `src/views/<page>/components/<Name>/`
```

**示例**：

- `AppHeader`（多处使用）→ `src/components/AppHeader/`
- `DashboardChart`（仅仪表盘使用）→ `src/views/dashboard/components/DashboardChart/`

**约束**：`src/components/` 中的组件必须是真正通用的，业务耦合组件禁止放入。

## 组件编写要点

- 组件库二次封装优先使用 Element Plus 组件
- 复杂组件优先使用 composable 抽取逻辑，保持模板简洁
- `v-model` 双向绑定遵循 `modelValue` + `update:modelValue` 规范
- 列表渲染必须提供唯一 `key`，禁止使用数组索引作为 key
