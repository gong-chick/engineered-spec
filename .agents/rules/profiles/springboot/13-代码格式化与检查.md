# 代码格式化与检查（Spring Boot）

## 工具链

| 工具 | 职责 | 配置文件 |
|------|------|----------|
| **Google Java Format** / **Spotless** | Java 代码格式化（缩进、括号等） | `pom.xml` / `build.gradle` 中的插件配置 |
| **Checkstyle** | 代码风格检查（命名、注释等） | `checkstyle.xml` |
| **SpotBugs** | 静态分析（潜在 bug 检测） | `spotbugs-exclude.xml` |

## 格式化规则

- 统一使用项目约定的 Java 格式化风格（Google Style 或 Alibaba Java Style）
- 缩进：4 个空格，禁止使用 Tab
- 单行最大宽度：120 字符
- 所有 `import` 按字母顺序排列，禁止 wildcard import（`import java.util.*`）

## Checkstyle 核心规则

- 类、方法、字段命名遵循 Java 命名约定
- 公共类和公共方法必须有 Javadoc 注释
- 禁止空 catch 块（必须有注释说明忽略原因）

## Maven / Gradle 集成

```xml
<!-- Maven 示例：Spotless 格式化插件 -->
<plugin>
  <groupId>com.diffplug.spotless</groupId>
  <artifactId>spotless-maven-plugin</artifactId>
  <configuration>
    <java>
      <googleJavaFormat/>
    </java>
  </configuration>
</plugin>
```

## 开发流程集成

- 提交前通过 Maven/Gradle 的 `verify` 阶段执行 Checkstyle 和 SpotBugs
- IDE 推荐安装 Checkstyle 插件，实时提示规范问题
- CI 流水线必须包含 `checkstyle:check` 步骤，失败则阻断合并
