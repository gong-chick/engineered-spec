# 分层规范（Spring Boot）

## 分层职责

| 层次 | 职责 | 禁止事项 |
|------|------|----------|
| Controller | 接收 HTTP 请求、参数校验、调用 Service、返回响应 | 禁止包含业务逻辑、禁止直接调用 Repository |
| Service | 业务逻辑编排、事务管理、跨模块调用 | 禁止直接处理 HTTP 请求 / 响应对象 |
| Repository | 数据访问、SQL / ORM 操作 | 禁止包含业务逻辑 |
| Domain / Entity | 领域模型定义 | 禁止依赖 Service 或 Repository |
| DTO | 数据传输对象，用于跨层传递 | 禁止包含业务逻辑 |

## 调用规则（NON-NEGOTIABLE）

- Controller 只调用 Service，不直接调用 Repository
- Service 调用 Repository 或其它 Service（同层可调用）
- Repository 只负责数据操作，不引用 Service

## 事务管理

- 事务注解 `@Transactional` 加在 Service 层
- 只读操作使用 `@Transactional(readOnly = true)` 提升性能
- 跨 Service 的复杂事务需明确传播行为（`REQUIRED` / `REQUIRES_NEW`）

## 接口与实现分离

- Service 层优先定义接口（`XxxService`），由 `XxxServiceImpl` 实现
- 单实现场景下可酌情省略接口，但需团队统一约定
