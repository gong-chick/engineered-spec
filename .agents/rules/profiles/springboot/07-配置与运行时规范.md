# 配置与运行时规范（Spring Boot）

## 配置文件分层

```text
src/main/resources/
├── application.yml          # 公共配置（不含敏感信息）
├── application-dev.yml      # 开发环境
├── application-test.yml     # 测试环境
├── application-prod.yml     # 生产环境
└── bootstrap.yml            # 配置中心加载（如使用）
```

- 通过 `spring.profiles.active` 激活环境配置
- 公共配置放 `application.yml`，环境差异放对应的 profile 文件

## 敏感配置管理（NON-NEGOTIABLE）

- 数据库密码、中间件密码、第三方 API 密钥禁止写入 Git
- 生产敏感配置通过配置中心（Nacos / Apollo）或环境变量注入
- 本地开发可使用 `.env` 文件（已加入 `.gitignore`）存储敏感值

## 配置类约定

- 自定义配置使用 `@ConfigurationProperties` 而非多处 `@Value`
- 配置类放在 `config/` 包，命名为 `XxxProperties` 或 `XxxConfig`
- 配置校验：在 `@ConfigurationProperties` 类上使用 `@Validated` 及 JSR-303 注解

## Bean 生命周期

- 使用构造函数注入（而非字段注入），便于测试与不变性保证
- 避免循环依赖；若存在，优先通过重构消除，而非使用 `@Lazy`
- `ApplicationRunner` / `CommandLineRunner` 仅用于应用启动后的一次性初始化
