# 代码格式化与检查（React）

## 工具链

| 工具 | 职责 | 配置文件 |
|------|------|----------|
| **Prettier** | 代码格式化（缩进、引号、分号等） | `.prettierrc.json` |
| **ESLint** | 代码质量与风格检查（未使用变量、潜在 bug 等） | `.eslintrc.cjs` / `.eslintrc.js` |
| **Stylelint** | 样式文件检查（CSS/SCSS/Less 规范） | `.stylelintrc.json` |

三者职责不重叠：Prettier 管格式，ESLint 管逻辑，Stylelint 管样式。

## 配置来源

- 项目根目录的 lint/format 配置文件由**规范库统一管理**
- `install.sh init` 时自动部署到目标项目，`update` 时同步更新
- 项目可在基础配置上**扩展**（如追加 i18n 插件），但不应修改基础规则

## Prettier 统一规则

| 配置项 | 值 | 说明 |
|--------|-----|------|
| `semi` | `true` | 语句末尾加分号 |
| `tabWidth` | `2` | 缩进 2 空格 |
| `singleQuote` | `true` | 使用单引号 |
| `printWidth` | `100` | 单行最大宽度 |
| `trailingComma` | `"es5"` | 对象/数组末尾加逗号（减少 diff 噪音） |
| `endOfLine` | `"auto"` | 自动适配换行符 |
| `arrowParens` | `"avoid"` | 单参数箭头函数省略括号 |
| `proseWrap` | `"never"` | Markdown 不自动换行 |

## ESLint 核心规则

### 通用基础

- 项目根据自身技术栈在 `extends` 中配置合适的共享规则集
- 项目可追加插件（如 `@intlify/vue-i18n`、`eslint-plugin-import` 等）

### Vue 项目

```js
// .eslintrc.cjs
module.exports = {
  root: true,
  // 项目按需配置 extends（如 eslint:recommended、plugin:vue/vue3-recommended 等）
};
```

### React 项目

```js
// .eslintrc.js
module.exports = {
  root: true,
  settings: {
    'import/resolver': {
      alias: [['@', './src']],
    },
  },
  // 项目按需配置 extends（如 eslint:recommended、plugin:react/recommended 等）
};
```

## Stylelint 核心规则

- 继承 `stylelint-config-standard` 基础规则
- Vue 项目追加 `stylelint-config-html/vue` 和 `stylelint-config-recommended-vue`
- 使用 `postcss-html` 作为自定义语法解析器（解析 `.vue` 文件中的 `<style>`）
- 忽略与业务组件库冲突的规则（如 `no-descending-specificity`）

## 开发流程集成

### IDE 保存时自动格式化（推荐）

项目可在 `.vscode/settings.json` 中配置：

```json
{
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": "explicit",
    "source.fixAll.stylelint": "explicit"
  }
}
```

### 提交前自动检查（husky + lint-staged）

使用 `husky` 注册 Git hooks，`lint-staged` 只检查暂存文件：

| 工具 | 职责 | 配置文件 |
|------|------|----------|
| husky | 注册 Git hooks（pre-commit、commit-msg） | `.husky/` |
| lint-staged | 对暂存文件执行 lint/format | `.lintstagedrc` |
| commitlint | 校验 commit message 格式 | `commitlint.config.js` |

**部署前提**：上述 `.husky/`、`.lintstagedrc`、`commitlint.config.js` 仅在执行 `install.sh` / `install.ps1` 时**选择安装提交校验**（`--husky` 或交互选 Y）后才会复制到目标项目并配合 `husky install` 注册 Git hooks。若选择跳过提交校验（`--no-husky` 或交互选 N），则不会下发这些文件；仅安装 lint/format 时只会得到 ESLint、Prettier、Stylelint 等配置。

**lint-staged 检查范围**：

```json
{
  "src/**/*.{vue,less,scss,css}": [
    "stylelint --fix"
  ],
  "src/**/*.{js,ts,jsx,tsx,json,vue}": [
    "eslint --fix"
  ]
}
```

### Commit Message 规范（Conventional Commits）

统一使用 [Conventional Commits](https://www.conventionalcommits.org/) 格式，由 commitlint 强制校验：

```
<type>(<scope>): <subject>
```

| type | 说明 |
|------|------|
| `feat` | 新功能 |
| `fix` | 修复 bug |
| `docs` | 文档变更 |
| `style` | 代码格式（不影响逻辑） |
| `refactor` | 重构（非新功能、非修复） |
| `test` | 测试相关 |
| `chore` | 构建、工具、依赖等杂项 |

**示例**：

```
feat(order): 新增工单批量导出功能
fix(auth): 修复 token 过期后未跳转登录页的问题
docs(api): 补充资产库接口类型定义
```

- `scope` 可选，建议填写业务模块名
- `subject` 使用中文，简明扼要

## 忽略文件约定

各工具的 ignore 文件遵循统一基线，至少忽略以下目录：

```
node_modules
dist
lib
assets
public
```

项目可根据实际情况追加特有的忽略路径。
