这是协议暂停入口，不是取消命令。

语义固定为：

- 暂停当前 run
- 保留当前产物、当前专家停点和门禁上下文
- 后续可通过 `/spec-continue` 或自然语言“继续 / 恢复执行”恢复

先执行：

```bash
./node_modules/.bin/ai-spec-auto protocol-stop --target . --json
```

然后：

1. 若当前没有活动 run 或当前已是终态，只输出当前状态说明
2. 若成功暂停，只告诉用户：
   - 当前已暂停
   - 当前停在哪个阶段
   - 如何恢复
3. 不要把 `/spec-stop` 解释成取消
4. 不要清理当前 OpenSpec 产物、运行态或上下文

对用户摘要保持简洁：

- 当前状态
- 当前停点
- 恢复方式
